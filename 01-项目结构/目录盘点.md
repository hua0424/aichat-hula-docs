# 目录盘点

## 结论
AIChat 当前采用多仓并行方式，核心交付链为客户端、管理端、服务端与 aichat-plugins 通讯插件四块；通过 hula-docs 统一治理。

## 根目录
`/home/hua/projects/aichat`

## 一级目录说明
- `HuLa`：即时通讯客户端（桌面/移动优先，含 Tauri）
- `HuLa-Admin`：后台管理端
- `HuLa-Server`：后端微服务
- `aichat-plugins`：aichat 与各种 openclaw 变体的通讯插件（替代原 openclaw 项目，仓库：https://github.com/hua0424/aichat-plugins.git）

## 核心仓库现状（已核验）
- 三仓均为有效 Git 仓库
- 分支：`master`
- 远程：`origin` 均可用
- Push 校验：`git push --dry-run origin HEAD` 正常

## 关键子目录（摘要）
### HuLa
- `src/`：前端主业务
- `src-tauri/`：桌面容器与 Rust 相关
- `docs/`：原项目文档
- `skills/`：扩展能力

### HuLa-Admin
- `src/`：管理端主业务
- `config/`：构建/环境配置
- `tests/`：测试代码

### HuLa-Server
- `luohuo-cloud/`：业务微服务集合（im/oauth/gateway/ai/...）
- `luohuo-util/`：基础能力与公共组件
- `docs/`：服务端文档
