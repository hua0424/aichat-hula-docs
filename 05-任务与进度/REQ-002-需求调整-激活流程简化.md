# REQ-002 需求调整：激活流程简化

> 需求文档：`04-需求与设计/需求/REQ-002-aiclaw-AI助理功能需求设计.md`（v1.2）
> 决策记录：`07-决策记录/ADR-001-aiclaw架构决策.md`（D15）
> 创建日期：2026-03-18
> 背景：联调中发现激活流程需要手动编辑 .env 配置 5 个参数，操作繁琐，简化为一条命令激活

---

## 变更概述

将原来的"手动配 .env → 启动 plugins"改为"一条命令激活"：

```
原流程：创建 → 拿到明文 token → 手动编辑 .env（5个参数）→ npm run dev
新流程：创建 → 拿到激活 token → aichat activate --backend openclaw --token <token> → 自动连接
```

核心改动：引入双 token 模型（激活 token + 连接 token），激活 token 为加密串（AES-256-GCM），包含 uid + 连接 token + timestamp。
