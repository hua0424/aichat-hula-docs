## 二、后端待办（按优先级）

## P0-1 CORS 跨域配置（必做）
- **Owner**：backend（峰哥）
- **状态**：✅ 已完成（2026-03-12）
- **内容**：
  - 允许前端域名跨域（开发：`http://localhost:5310`；生产：正式域名）
  - 允许请求头：`Content-Type`、`Authorization`
  - 允许方法：`GET/POST/PUT/DELETE/OPTIONS`
  - 开发可临时放宽，生产需按域名白名单收敛
- **影响范围**：全部 HTTP API + WebSocket 握手
- **验收标准**：浏览器 Network 中无 CORS 拦截，预检 OPTIONS 返回正常。

## P0-2 WebSocket 握手 Token 方案（必做）
- **Owner**：backend（峰哥）
- **状态**：✅ 已完成（2026-03-12）
- **内容**：明确 Web 端 WS 鉴权 token 的传递方式（浏览器不能自定义 WS header）：
  1. URL query：`ws://host:18080/api/ws/ws?token=xxx&clientId=yyy`（推荐）
  2. `Sec-WebSocket-Protocol`
  3. 首帧消息认证
- **前端建议**：方案 1，兼容性和实现成本最佳。
- **验收标准**：给出唯一生效方案 + 示例 URL/示例报文，前端可稳定连通。

- **Owner**：backend（峰哥）
- **状态**：✅ 已完成（2026-03-12）
- **内容**：
  - 明确 API base path 前缀（如 `/capi/`、`/api/`、`/v1/`）
  - 提供登录接口完整路径（示例：`POST http://host:18080/.../login`）

- **验收标准**：后端给出接口前缀与至少登录/刷新 token 的完整路径说明。
