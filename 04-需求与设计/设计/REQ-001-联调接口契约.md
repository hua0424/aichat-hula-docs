## 一、后端访问地址

团队容器均在同一 Docker 宿主机，通过 `aichat-dev-net` 内网互通，直接用容器名访问即可。

| 访问方 | API Base | WebSocket |
|--------|---------|-----------|
| **容器内**（推荐） | `http://hula-server-dev:18760/api` | `ws://hula-server-dev:18760/api/ws/ws` |
| **宿主机**（Linux） | `http://172.18.0.3:18760/api` | `ws://172.18.0.3:18760/api/ws/ws` |

> 容器名 `hula-server-dev` 在 Docker 内网（`aichat-dev-net`）中始终可解析，比 IP 更稳定；宿主机 IP `172.18.0.3` 在容器重建后可能变化，以 `docker inspect hula-server-dev` 确认。

### `/api` 前缀说明

`/api` 由网关 `ContextPathFilter` 透明剥除后路由，**前端所有请求路径均以 `/api` 开头**，后端业务接口路径无需带 `/api`。

## 二、HTTP API 接口

### 2.1 登录

```
POST /api/oauth/anyTenant/login
Content-Type: application/json
Authorization: bHVvaHVvX3dlYjpsdW9odW9fd2ViX3NlY3JldA==
```

> `Authorization` 头是客户端凭证，固定值，Base64 编码的 `luohuo_web:luohuo_web_secret`（来自数据库 `def_client` 表）。

**请求体：**
```json
{
  "account": "string",
  "password": "string",
  "clientId": "string",
  "systemType": 2,
  "deviceType": "PC",
  "grantType": "PASSWORD"
}
```

| 字段 | 说明 |
|------|------|
| `clientId` | 客户端设备指纹，必填，WS 连接时也需携带 |
| `systemType` | 固定填 `2`（IM 聊天系统）|
| `deviceType` | `PC` 或 `MOBILE` |
| `grantType` | `PASSWORD`（密码登录）/ `CAPTCHA`（验证码）/ `MOBILE`（手机号）|

**成功响应 `data` 字段：**
```json
{
  "uid": 123456,
  "token": "<TOKEN_VALUE>",
  "refreshToken": "<REFRESH_TOKEN>",
  "expire": 2592000,
  "expiration": "2026-04-10T10:30:00",
  "client": "PC"
}
```

### 2.2 刷新 Token

```
POST /api/oauth/anyTenant/refresh
Content-Type: application/json
```

**请求体：**
```json
{
  "refreshToken": "string"
}
```

**成功响应**：同登录响应，返回新 `token` 和 `refreshToken`。

### 2.3 携带 Token（HTTP 请求）

所有需要鉴权的接口，在请求 Header 中携带：

```
Token: <TOKEN_VALUE>
```

### 2.4 统一响应体格式

```json
{
  "code": 200,
  "success": true,
  "msg": "",
  "data": { },
  "path": "/...",
  "version": "3.0.0",
  "timestamp": 1710226200000
}
```

| code | 含义 |
|------|------|
| `200` | 成功 |
| `406` | Token 已过期（HTTP 状态码仍为 200，需判断 `code` 字段）|
| 其他 | 业务错误，见 `msg` |

## 三、WebSocket 连接

### 3.1 连接地址

```
ws://172.18.0.3:18760/api/ws/ws?token=<TOKEN_VALUE>&clientId=<FINGERPRINT>
```

| 参数 | 说明 |
|------|------|
| `token` | 登录响应中 `data.token` 的值（小写 `token`）|
| `clientId` | 与登录时一致的设备指纹 |

### 3.2 说明

- 浏览器 WebSocket API 无法自定义 Header，token 通过 URL query 传递（已在网关实现 fallback）
- 无 token 或 token 无效时，连接会被服务端主动关闭
- 原有桌面客户端的 `Token` Header 方式不受影响，两种方式并存

### 3.3 路径说明

```
/api/ws/ws
  └─ /api     → 网关 ContextPathFilter 剥除
  └─ /ws      → 网关路由到 luohuo-ws 服务
  └─ /ws      → ws 服务内部 Handler 注册路径
```

## 四、CORS 配置

已在网关配置白名单，开发环境放行 `http://localhost:5310`：

| 响应头 | 值 |
|--------|-----|
| `Access-Control-Allow-Origin` | `http://localhost:5310` |
| `Access-Control-Allow-Methods` | `GET, POST, PUT, DELETE, OPTIONS` |
| `Access-Control-Allow-Headers` | `Content-Type, Authorization, Token` |
| `Access-Control-Allow-Credentials` | `true` |

如需增加允许来源，在 Nacos `common.yml` 中修改（逗号分隔）：
```yaml
luohuo:
  cors:
    allowed-origins: http://localhost:5310,https://your-domain.com
```
