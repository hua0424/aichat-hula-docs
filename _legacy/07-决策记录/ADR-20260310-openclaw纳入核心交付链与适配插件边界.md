# ADR-20260310-openclaw纳入核心交付链与适配插件边界

- 状态：Accepted
- 日期：2026-03-10
- 决策人：华血
- 参与方：manager / backend / devops / qa / frontend

## 背景
AIChat 项目现有文档将 openclaw 归类为工具链，不在本次核心交付链。随着业务推进，项目已明确需要开发 openclaw 与 HuLa-Server 的连接与通讯适配插件，若不纳入核心交付链，将导致职责边界、资源投入、测试验收与发布口径不一致。

## 备选方案
1. 维持原定义：openclaw 不纳入核心交付链，仅作为外围工具。
2. 纳入核心交付链：将 openclaw 适配插件作为 AIChat 本期正式交付内容。

## 决策
采用方案 2：openclaw 纳入本期核心交付链。

交付边界定义如下：
- openclaw 侧：提供标准能力接口与插件扩展点。
- 适配插件侧（本次交付重点）：负责 openclaw 与 HuLa-Server 的连接、协议转换、消息路由、状态同步、错误映射。
- HuLa-Server 侧：提供可被适配层消费的服务接口与鉴权策略。

## 决策依据
- 业务上已存在明确的跨系统通讯需求，且属于主链路能力。
- 将适配层纳入核心交付可确保开发、测试、发布按统一门禁执行。
- 便于风险前置管理（兼容性、性能、稳定性）与责任闭环。

## 影响
- 正向：
  - 统一交付口径，避免 openclaw 相关需求“游离于主流程”
  - 明确 Owner 与验收路径，提升跨仓协作效率
  - 便于纳入压测、回归与发布清单
- 负向：
  - 本期范围扩大，开发与联调复杂度上升
- 风险与缓解：
  - 风险：协议不一致导致联调阻塞
  - 缓解：先产出接口契约、时序图与错误码映射，再编码
