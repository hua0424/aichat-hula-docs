Authorization: bHVvaHVvX3dlYjpsdW9odW9fd2ViX3NlY3JldA==

```json

  "systemType": 2,
  "deviceType": "PC",

```

成功后返回 `data.token` 和 `data.uid`。

### 2.2 携带 Token

所有鉴权接口在 Header 中加：

```
Token: <TOKEN_VALUE>
ApplicationId: 1
```

### 2.3 WebSocket 连接

```
ws://localhost:18080/api/ws/ws?token=<TOKEN>&clientId=<FINGERPRINT>
```

- `token`：登录返回的 `data.token`
- `clientId`：与登录时一致的设备指纹

## 三、REQ-002 Aiclaw 新增接口

### 3.1 HTTP API

所有接口需要 owner 登录态（`Token` + `ApplicationId` Header）。

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/im/aiclaw/create` | 创建 AI 助理 |
| GET | `/api/im/aiclaw/list` | 获取 AI 助理列表 |
| PUT | `/api/im/aiclaw/{uid}/profile` | 修改资料 |
| POST | `/api/im/aiclaw/{uid}/reset-token` | 重置 Token |
| POST | `/api/im/aiclaw/{uid}/deactivate` | 停用 |
| POST | `/api/im/aiclaw/{uid}/restore` | 恢复 |
| POST | `/api/im/aiclaw/{uid}/auth-confirm` | 机器码授权确认 |

#### 创建 AI 助理

```
POST /api/im/aiclaw/create
```
```json
{ "name": "我的AI助理", "avatar": "", "description": "简介" }
```
响应：
```json
{
  "success": true,
  "data": {
    "uid": "140068040294400",
    "name": "我的AI助理",
    "token": "7cc62771-4471-451a-916b-9c0a8bd43ebc"
  }
}
```
> `token` 仅返回一次，前端需弹 Dialog 展示并提供复制按钮。

#### 列表查询

```
GET /api/im/aiclaw/list
```
响应 `data` 为数组：
```json
[{
  "uid": "140068040294400",
  "name": "我的AI助理",
  "avatar": "",
  "description": "简介",
  "authStatus": 0,
  "adapterType": "openclaw",
  "createTime": 1773721225000
}]
```

`authStatus` 取值：`0`=未激活、`1`=已激活、`2`=已停用。

### 3.2 WS 新增事件类型

#### Server → Client 推送

| type (String) | 说明 | data 结构 |
|----------------|------|-----------|
| `streamStart` | 流式消息开始 | `{ msgId, fromUid, toUid, roomId }` |
| `streamDelta` | 流式消息片段 | `{ msgId, chunk, seq }` |
| `streamEnd` | 流式消息结束 | `{ msgId, fullContent, status }` |
| `aiclawAuthRequest` | 机器码授权请求 | `{ aiclawUid, aiclawName, oldMachineCode, newMachineCode }` |

#### 前端处理逻辑

```
收到 streamStart → chatStore.pushMsg({ content:'', streaming:true, tempMsgId:msgId })
收到 streamDelta → chatStore.appendStreamContent(msgId, chunk)  // rAF 节流
收到 streamEnd   → chatStore.updateMsg(msgId, { content:fullContent, streaming:false })
```

- `status: "complete"` — 正常完成，`fullContent` 已落库
- `status: "error"` — 失败，`fullContent` 为错误提示，不落库

### 3.3 用户类型判断

```typescript
// 判断是否为 AI 助理
if (fromUser.userType === 4) {
  // 显示 AI 标识、顶部提示条等
}
```

`user_type` 枚举：`1`=系统用户、`2`=机器人、`3`=普通用户、**`4`=AI助理**
