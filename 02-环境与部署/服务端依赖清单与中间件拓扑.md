### 4.3 数据流路径

```
客户端 ──WS──→ Gateway(:18760) ──路由──→ WS-Server(:18762)
                                          │
                                          ├─ 在线状态 → Redis
                                          ├─ 消息入队 → RocketMQ (FRONTEND_MSG_INPUT_TOPIC)
                                          │
                                          ↓
                                    IM-Server(:18763)
                                          │
                                          ├─ 消息存库 → MySQL
                                          ├─ 消息推送 → RocketMQ (MSG_PUSH_OUTPUT_TOPIC)
                                          │
                                          ↓
                                    WS-Server(:18762)
                                          │
                                          ├─ 本节点 → SessionManager 直推
                                          └─ 跨节点 → RocketMQ (PUSH_TOPIC + nodeId)
```

## 五、RocketMQ Topic 清单

| Topic | 用途 | 生产者 | 消费者 |
|-------|------|--------|--------|
| `FRONTEND_MSG_INPUT_TOPIC` | 前端消息入队 | WS-Server | IM-Server |
| `MSG_PUSH_OUTPUT_TOPIC` | 消息推送出队 | IM-Server | WS-Server |
| `MSG_PUSH_ACK_TOPIC` | 消息确认 | WS-Server | IM-Server |
| `MSG_PUSH_READ_TOPIC` | 消息已读 | WS-Server | IM-Server |
| `PUSH_TOPIC` (websocket_push) | 跨节点 WS 推送 | WS-Server | WS-Server(其他节点) |
| `PUSH_DELAY_TOPIC` | 延迟重试推送 | WS-Server | WS-Server |
| `LOGIN_MSG_TOPIC` | 登录消息 | OAuth-Server | WS-Server |
| `SCAN_MSG_TOPIC` | 扫码消息 | OAuth-Server | WS-Server |

## 七、构建产物

```
HuLa-Server/luohuo-cloud/
├── luohuo-gateway/luohuo-gateway-server/target/luohuo-gateway-server.jar
├── luohuo-oauth/luohuo-oauth-server/target/luohuo-oauth-server.jar
├── luohuo-ws/luohuo-ws-server/target/luohuo-ws-server.jar
└── luohuo-im/luohuo-im-server/target/luohuo-im-server.jar
```

Runtime 容器启动时将上述 4 个 JAR 复制到 `/runtime/jars/`，通过 JVM 参数 `-Dserver.port` 分配端口。
