# 环境登记与责任分工（v1）

## 结论
先按“角色分责 + 统一模板登记 + manager汇总评估 + 华血确认后执行”的节奏推进。

## 统一登记字段（所有成员必须填写）
- 工作目录（本地 clone 路径）
- Dockerfile 路径与基础镜像
- 构建命令 / 测试命令
- 运行端口（容器内/宿主机）
- 依赖服务（DB/Redis/MQ/网关等）
- 环境变量清单（仅键名，不填密钥）
- 健康检查方式
- 镜像命名与 tag 规范

## 分工
### 1) 前端（HuLa + HuLa-Admin）
- Owner：frontend（阿飞）
- 输出：
  - `HuLa` 与 `HuLa-Admin` 各自 Dockerfile 草案
  - 前端构建产物目录与 Nginx 运行方案
  - 前端服务端口与反向代理需求
- 截止：T+1 天

### 2) 后端（HuLa-Server + aichat-plugins）
- Owner：backend（峰哥）
- 输出：
  - 服务端构建镜像方案（JDK21 + Maven）
  - aichat-plugins 通讯插件容器化开发/运行方案
  - 微服务运行清单（gateway/oauth/im/ai/ws等）
  - 中间件依赖清单（MySQL/Redis/RocketMQ/Nacos等）
- 截止：T+1 天

### 3) 运维（测试环境）
- Owner：devops（石哥）
- 输出：
  - `docker-compose` 基础编排草案
  - 网络、卷、日志、重启策略
  - 测试环境部署与回滚步骤草案
- 截止：T+1 天

### 4) 测试（验收）
- Owner：qa（思思）
- 输出：
  - 测试环境可用性验收清单
  - 冒烟路径与回归范围
  - 缺陷登记模板
- 截止：T+2 天
