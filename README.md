# hula-docs

AIChat（基于 HuLa）项目统一文档仓库。

## 目标
- 让新成员 **30 分钟内**了解项目全貌（目录、技术栈、环境、启动方式）
- 统一管理：需求、设计、任务、进度、问题、决策、发布
- 降低信息分散导致的沟通与交付风险

## 当前项目基线（已核实）
- 根目录：`/home/hua/projects/aichat`
- 子项目：
  - `HuLa`：客户端（Vue3 + Vite7 + Tauri + TS）
  - `HuLa-Admin`：管理端（Vue3 + Vite + TS）
  - `HuLa-Server`：服务端（Spring Cloud 2024 + Spring Boot3 + Java21 + Maven）
  - `aichat-plugins`：aichat 与各种 openclaw 变体的通讯插件（仓库：https://github.com/hua0424/aichat-plugins.git）

## 建议文档使用原则
1. **结论先行**：每个文档开头写「结论/状态」
2. **单一事实源**：同类信息只放一个地方，其他文档链接引用
3. **可追溯**：重大结论必须在 `07-决策记录` 留痕
4. **可执行**：任务必须有 owner、输入输出、截止时间

## 目录导航
- `00-总览`：项目总览、里程碑、角色分工
- `01-项目结构`：代码仓结构、模块边界、依赖关系
- `02-环境与部署`：环境清单、部署架构、配置说明
- `03-开发规范`：分支/提交流程、代码规范、联调约定
- `04-需求与设计`：需求规格、架构设计、接口契约
- `05-任务与进度`：任务看板、周报、燃尽
- `06-会议与讨论`：会议纪要、讨论结论
- `07-决策记录`：ADR（架构/技术决策记录）
- `08-风险与问题`：风险台账、阻塞问题追踪
- `09-运维与发布`：发布计划、回滚预案、值班与告警
- `10-附录`：术语、FAQ、外部链接
- `.templates`：模板库（新文档直接套模板）

## 推荐协作节奏
- 每日：更新 `05-任务与进度/看板.md`
- 每次评审后：更新 `04-需求与设计` + `07-决策记录`
- 每次发布前后：更新 `09-运维与发布`

## 快速入口
- 项目结构：`01-项目结构/目录盘点.md`
- 环境状态：`02-环境与部署/环境基线.md`
- 看板：`05-任务与进度/看板.md`
- 风险：`08-风险与问题/风险清单.md`
