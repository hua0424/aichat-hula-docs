# ADR-20260309-开发测试发布流程与分支策略

- 状态：Accepted
- 日期：2026-03-09
- 决策人：华血
- 参与方：manager / devops / qa / frontend / backend

## 背景
AIChat（HuLa）项目需建立统一、可追溯、低风险的开发-测试-发布流程，并确保环境隔离与职责清晰。

## 决策
1. 采用 Docker 化开发与测试部署流程：开发提交代码，CI 构建镜像，运维手动审批发布，QA验收。
2. 当前镜像仓库先使用本地镜像存储，后续按条件迁移到集中仓库。
3. 使用分支策略：`main`（生产）/`develop`（测试基线）/`feature/*`（开发）。

## 决策依据
- 提高可复现性与可回滚性
- 避免测试环境源码构建导致的环境漂移
- 保证职责边界和审计链路清晰

## 影响
- 正向：流程标准化、风险下降、问题定位效率提升
- 负向：前期流程与CI配置成本上升
- 风险缓解：以最小流程先落地，逐步完善自动化
