## 二、镜像与容器规划

### 1) Builder 容器
1. `hula-web-builder`
   - Dockerfile: `infra/docker/Dockerfile.hula.web`
   - 基础镜像: `node:24-bookworm-slim`
   - 命令: `pnpm install && pnpm build`
   - 产物输出: `artifacts/hula/web`

2. `hula-admin-builder`
   - Dockerfile: `infra/docker/Dockerfile.admin.web`
   - 基础镜像: `node:24-bookworm-slim`
   - 命令: `pnpm install && pnpm run build:prod`
   - 产物输出: `artifacts/admin/web`

3. `hula-desktop-linux-builder`
   - Dockerfile: `infra/docker/Dockerfile.hula.desktop-linux`
   - 基础镜像: `rust:1.82-bookworm`（内置 Node24 + pnpm10）
   - 命令: `pnpm tauri build --bundles deb,appimage`
   - 产物输出: `artifacts/hula/desktop-linux`

4. `hula-android-builder`
   - Dockerfile: `infra/docker/Dockerfile.hula.android`
   - 基础镜像: `mingc/android-build-box:latest`
   - 命令: `pnpm tauri android build`
   - 产物输出: `artifacts/hula/android`

### 2) Runtime 容器
1. `hula-web-runtime`
   - 镜像: `nginx:1.27-alpine`
   - 端口: `8081:80`
   - 静态目录: `artifacts/hula/web`

2. `hula-admin-runtime`
   - 镜像: `nginx:1.27-alpine`
   - 端口: `8082:80`
   - 静态目录: `artifacts/admin/web`

## 四、构建与测试命令

```bash
cd /home/hua/projects/frontend_workspace/aichat/infra
cp .env.example .env

# HuLa Web 构建
docker compose --profile build-web run --rm hula-web-builder

# HuLa-Admin Web 构建
docker compose --profile build-admin run --rm hula-admin-builder

# HuLa Linux Desktop 构建
docker compose --profile build-desktop-linux run --rm hula-desktop-linux-builder

# HuLa Android 构建
docker compose --profile build-android run --rm hula-android-builder

# Web 运行验证
docker compose --profile run-web up -d hula-web-runtime
docker compose --profile run-admin up -d hula-admin-runtime
```

### 团队需关注事项（共享信息）
- Android 基础镜像 `mingc/android-build-box:latest` 层很大，首次拉取耗时长且可能出现 EOF/short read，建议在 CI/构建机做预拉取与缓存。
- Linux 容器对大小写敏感，前端资源路径和文件名必须严格一致（已在 Admin 出现过一次实际问题）。
