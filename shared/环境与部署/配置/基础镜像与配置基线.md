# 基础镜像与配置基线（最终版）

## 结论
构建统一采用容器，宿主机不安装构建工具链。后端基线固定 **JDK 21**，不使用 JDK 11。

---

## 1. 基线约束
- 宿主机要求：Docker / Docker Compose / Git
- 构建方式：源码目录挂载进构建容器
- 环境配置：`.env` 分层（dev/staging/prod）
- 镜像存储：本地私有仓库

---

## 2. 基础镜像（Builder / Runtime）

## 2.1 HuLa（客户端）
- Builder：`node:24-bookworm-slim` + `pnpm@10`
- Runtime：`nginx:alpine`

## 2.2 HuLa-Admin（管理端）
- Builder：`node:24-bookworm-slim` + `pnpm@10`
- Runtime：`nginx:alpine`

## 2.3 HuLa-Server（服务端）
- Builder：`maven:3.9-eclipse-temurin-21`
- Runtime：`eclipse-temurin:21-jre`

> 说明：JDK11 不满足当前后端技术栈基线，不纳入支持范围。

---

## 3. Dockerfile建议模板（摘要）

## 3.1 前端（HuLa/HuLa-Admin）
```dockerfile
# Builder
FROM node:22-bookworm AS builder
WORKDIR /app
RUN corepack enable && corepack prepare pnpm@10 --activate
COPY . .
RUN pnpm install --frozen-lockfile && pnpm build

# Runtime
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
```

## 3.2 后端（HuLa-Server）
```dockerfile
# Builder
FROM maven:3.9-eclipse-temurin-21 AS builder
WORKDIR /build
COPY . .
RUN mvn -B -ntp clean package -DskipTests

# Runtime
FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=builder /build/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","/app/app.jar"]
```

## 4. `.env` 分层规范

## 4.1 文件结构
- `.env.common`（公共非敏感）
- `.env.dev`
- `.env.staging`
- `.env.prod`

## 4.2 示例字段（按服务扩展）
```env
# common
TZ=Asia/Shanghai
LOG_LEVEL=INFO

# server
SERVER_PORT=8080
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=aichat
DB_USER=aichat
DB_PASSWORD=***

# frontend/admin
API_BASE_URL=http://aichat-server:8080
```

## 4.3 规则
- 禁止将真实生产密钥写入仓库
- 测试/生产凭据通过部署侧注入
- 所有环境变量清单在文档中维护字段说明

## 5. 风险提示
- HuLa 若进行桌面（Tauri）打包，需要单独桌面构建镜像与系统依赖，不与 Web 构建链路混用。
- 建议“Web构建链路”和“桌面构建链路”独立维护。
